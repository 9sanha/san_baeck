git 기초

커밋 (commit) : 파일을 추가하거나 변경 내용을 저장소에 저장 (이력을 남기는 작업이기에 커밋 메시지 입력 필수)
  -> 커밋 메시지 예시 : line 1 : 커밋 내 변경 내용 요약; line 2 : 빈 칸; line 3 : 변경한 이유 
  
푸시 (push) : 파일을 추가하거나 변경 내용을 원격 저장소에 업로드 

저장소 : 로컬 저장소와 원격 저장소로 구성됨, 보통 로컬 저장소에서 작업 수행 후 수행결과를 원격 저장소에 저장 (원격 저장소는 코드를 공개하고 싶을 때 업로드)

브랜치 (branch): 한 소프트웨어 안에서 여러 개발자들이 동시에 다양한 작업을 할 수 있게 만들어주는 기능

사용법
- 작은 작업 단위로 커밋하고 어느정도 작업이 끝나갈 때 푸시
- 커밋 메시지는 로그를 따라가는 데에도 도움이 됨
